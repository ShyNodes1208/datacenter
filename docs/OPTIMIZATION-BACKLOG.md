## 设备落位图

### OPT-001：合并单元格视觉还原度
- **日期**：2026-07-23
- **优先级**：中
- **模块**：前端 RackDeviceView
- **现状**：合并单元格的多 U 设备已能正确合并显示（蓝色块跨越多行），但视觉上未完全还原 Excel 效果（无边框融合、文字未垂直居中）
- **方向**：后续考虑用 CSS `border-collapse` 或更精确的 grid 布局还原合并单元格的视觉效果

### OPT-002：单柜导入与批量导入代码重复
- **日期**：2026-07-23
- **优先级**：低
- **模块**：后端 DevicePositionsController + DevicePositionsBatchController
- **现状**：两个 Controller 中解析 Excel 行、检测合并单元格、构建 DevicePosition 的逻辑几乎相同
- **方向**：提取共享的 Excel 解析服务，消除重复

### OPT-003：机柜编号匹配使用 Contains 可能误匹配
- **日期**：2026-07-23
- **优先级**：中
- **模块**：后端 DevicePositionsController + DevicePositionsBatchController
- **现状**：机柜编号匹配使用 `normalized.Contains(rack.Code)`，如果一个机柜编号是另一个的前缀（如 `2-2[06]` 和 `2-2[06]-B`），短的会误匹配长的表头
- **方向**：改用更精确的匹配逻辑（正则、单词边界），当前 2 个机柜无此问题

### OPT-004：导入前缺少预览步骤
- **日期**：2026-07-23
- **优先级**：中
- **模块**：前端 RackDeviceView + HomeView
- **现状**：选择文件后直接执行导入，用户无法预览将要导入的内容。设备落位是全量覆盖，选错文件会立即覆盖已有数据
- **方向**：增加预览步骤（显示即将导入的设备列表），用户确认后再写入

### OPT-005：机柜卡片缺少占用统计
- **日期**：2026-07-23
- **优先级**：低
- **模块**：前端 HomeView + 后端 GET /api/racks
- **现状**：机柜卡片只显示编号和 U 位数，未显示已用/空闲 U 数
- **方向**：扩展 GET /api/racks 返回 device-position 统计，或单独加轻量统计端点

### OPT-006：DB 缺少 UNumber >= 1 的 CHECK 约束
- **日期**：2026-07-23
- **优先级**：低
- **模块**：后端 DevicePosition 模型 + Migration
- **现状**：UNumber 的有效性只在应用层校验，数据库层没有 CHECK 约束
- **方向**：添加 `CK_DevicePositions_UNumber` CHECK 约束
