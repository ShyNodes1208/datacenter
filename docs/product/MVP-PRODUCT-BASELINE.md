## 2. 产品背景

hangyu 提出开发一套企业机房服务器落位可视化系统。当前企业机房中的服务器位置管理依赖人工记录、表格和个人经验，缺乏统一、直观、可靠的位置管理工具。需要一套以二维机房平面图为主要操作入口的系统，实现机房空间、机柜和设备的落位管理。

本系统以二维机房平面图为日常操作入口，以单机柜局部三维为辅助展示方式。但三维能力不属于本次 MVP 范围。

## 3. 当前问题

1. **位置记录不统一**：服务器具体位于哪个机房、哪个机柜、哪个 U 位，缺乏统一的系统记录。
2. **机柜空间不直观**：机柜空闲空间和已使用空间无法直观查看，依赖人工估算。
3. **位置变更难追踪**：服务器上架、移动和下架后，位置变化缺乏系统记录，难以追溯历史。
4. **查询效率低**：查询某台服务器的当前位置时依赖表格查找、人工记忆或询问特定同事。
5. **数据一致性差**：多人维护时容易出现记录不一致、U 位重复占用或信息过期。
6. **缺乏可视化展示**：无法通过网页二维机柜视图直观查看服务器在机柜中的位置。

## 4. 产品目标

MVP 产品核心目标：**建立一个可查、可视、可追溯的服务器落位管理工具。**

### 4.1 核心目标

1. **能查到**：记录机房、机柜和服务器基础信息，支持按关键信息查询服务器位置。
2. **能看懂**：使用二维机柜视图展示机柜 U 位占用情况，空闲和已占用一目了然。
3. **能正确上架**：支持服务器上架，校验 U 位可用性，防止重复占用。
4. **能正确移动**：支持服务器在机柜内或跨机柜移动，释放原位置并占用新位置。
5. **能正确下架**：支持下架操作，释放 U 位并保留操作记录。
6. **不会发生 U 位重复占用**：上架前强制校验目标 U 位可用性，系统层面保证同一 U 位在同一时间只能被一台在架服务器占用。
7. **位置变化可追踪**：所有位置变化均产生操作记录，可事后查询追溯。

### 4.2 非目标

本 MVP 明确不实现：

- 三维机房漫游或三维服务器模型
- 服务器运行状态实时监控和自动采集
- 完整的 CMDB（配置管理数据库）
- 网络设备和链路拓扑
- 电力、制冷、温湿度等环境数据
- 自动发现和自动录入服务器
- 资产全生命周期管理（采购、折旧、报废）
- 工单系统和审批流
- 完整的 RBAC 权限平台
- LDAP、AD 或单点登录
- 告警和通知平台
- 容量趋势预测和 AI 分析
- 报表中心和仪表盘
- 移动端 App
- 多租户支持
- 插件系统
- 微服务架构

## 5. MVP 成功标准

以下列出的每一条都是 MVP 可独立判断成功或失败的可观察标准：

| 编号 | 成功标准 | 判断方式 |
|---|---|---|
| SC-01 | 用户可以创建、查看和编辑机房信息 | 创建机房后可在机房列表中查看；编辑后信息正确更新 |
| SC-02 | 用户可以创建、查看和编辑机柜信息，包括配置 U 位总数 | 创建机柜时可指定 U 位总数；编辑后信息正确更新 |
| SC-03 | 用户可以录入服务器基础信息，服务器可以先录入但暂不上架 | 录入的服务器出现在服务器列表中，位置状态为"未上架" |
| SC-04 | 用户可以将未上架的服务器上架到指定机柜的指定 U 位范围 | 上架后服务器状态变为"在架"，目标 U 位显示为已占用 |
| SC-05 | 上架时系统拒绝超出机柜范围或已被占用的 U 位 | 输入无效目标位置时，系统给出明确拒绝提示，操作不生效 |
| SC-06 | 用户可以将已在架的服务器移动到新的机柜和 U 位 | 移动后原 U 位释放、新 U 位被占用，原位置记录可查看 |
| SC-07 | 用户可以将已在架的服务器下架，释放 U 位但服务器记录保留 | 下架后 U 位变为空闲，服务器记录存在于系统，位置状态变为"已下架" |
| SC-08 | 二维机柜视图正确展示所有 U 位的空闲或占用状态 | 空闲 U 位与已占用 U 位有明显视觉区分，已占用 U 位显示服务器名称 |
| SC-09 | 机柜容量统计与实际占用一致 | U 位总数、已用数、空闲数和使用率与实际状态匹配 |
| SC-10 | 用户可以通过服务器名称、管理 IP 等关键信息查询服务器 | 输入查询条件后返回匹配的服务器列表 |
| SC-11 | 位置变化产生操作记录，包含操作类型、前后位置、操作人和时间 | 操作记录页面可按服务器查看完整的位置变化历史 |
| SC-12 | 未登录用户或只读用户不能执行位置修改操作 | 尝试执行上架、移动、下架等操作时被拒绝 |

## 6. 产品范围

### 6.1 最小实现范围

MVP 必须实现以下能力：

1. 机房基础信息管理（创建、查看、编辑）
2. 机柜基础信息管理（创建、查看、编辑，含 U 位总数配置）
3. 机柜二维 U 位视图展示（空闲/已占用区分）
4. 服务器基础信息录入（可先录入后上架）
5. 服务器上架（U 位校验、占用）
6. 服务器移动（跨机柜支持、原位置释放）
7. 服务器下架（U 位释放、服务器记录保留）
8. 服务器查询（按名称、IP、状态等条件筛选）
9. 服务器与机柜占用关系展示
10. 机柜容量和使用情况展示（总数、已用、空闲、使用率）
11. 最小业务状态展示（机房状态、机柜状态、服务器位置状态、运行状态标记）
12. 最小操作记录（操作类型、前后位置、操作人、时间）
13. MVP 页面清单（9 个页面）

### 6.2 明确不实现范围

以下内容不属于 MVP，不得在当前阶段开发：

- 三维相关：3D 机房漫游、3D 服务器模型、Three.js 场景
- 实时监控：服务器运行状态自动采集、实时 CPU/内存/磁盘监控
- 完整 CMDB：自动发现、自动录入、资产生命周期、采购、折旧
- 工单审批：工单系统、审批流、变更管理
- 权限平台：RBAC、LDAP、AD、SSO、字段权限、菜单权限
- 环境数据：PDU、电力、温度、湿度、制冷
- 网络拓扑：网络设备管理、链路拓扑、端口管理
- 报表仪表：报表中心、驾驶舱、大屏首页、图表中心
- 移动端：移动 App、响应式移动端适配
- 高级能力：AI 预测、批量导入导出增强、多机房集团视图、多租户、插件系统

## 7. 用户角色

### 7.1 角色定义

MVP 只定义以下四个最小角色，不建立复杂权限矩阵和 RBAC 体系：

| 角色 | 职责 | 可执行操作 |
|---|---|---|
| 机房管理员 | 维护机房、机柜和服务器位置信息；执行上下架操作 | 所有操作：创建/编辑机房和机柜、录入服务器、上架、移动、下架、查看所有信息 |
| 运维人员 | 日常查询和位置操作 | 查看所有信息；执行上架、移动和下架操作（MVP 中与机房管理员具有相同的修改权限） |
| DBA/应用运维人员 | 查询服务器的位置和基础信息 | 查看所有服务器的信息（MVP 不实现按负责人过滤）；只读查询 |
| 只读查看人员 | 只能查看，不能修改 | 查看机房、机柜、服务器和二维视图；不能执行任何修改操作 |

### 7.2 角色约束

- MVP 不实现 RBAC 权限模型、菜单权限、字段权限。
- MVP 不实现组织架构继承或数据权限树。
- MVP 不实现 LDAP、AD 或单点登录。
- 所有页面访问均要求登录身份。MVP 不实现匿名访问。只读查看人员也必须登录后才能查看。
- 角色的操作权限差异必须在用户界面可观察（如不可执行的操作入口不可点击或被系统拒绝），具体实现方式由架构任务决定。

## 8. 核心业务对象

本章只定义业务对象的含义和最小业务字段。不设计数据库表结构、字段类型、索引、外键或物理模型。

### 8.1 机房

机房是服务器和机柜的物理空间容器。

| 业务字段 | 含义 | 必填 | 说明 |
|---|---|---|---|
| 机房标识 | 系统内唯一标识 | 是 | 由系统生成 |
| 机房名称 | 机房业务名称 | 是 | 全局唯一 |
| 位置说明 | 机房所在楼层、区域或地址 | 否 | 用于帮助定位机房物理位置 |
| 状态 | 启用或停用 | 是 | 默认启用 |
| 备注 | 补充说明 | 否 | |

### 8.2 机柜

机柜是机房内容纳服务器的物理柜体。

| 业务字段 | 含义 | 必填 | 说明 |
|---|---|---|---|
| 机柜标识 | 系统内唯一标识 | 是 | 由系统生成 |
| 机柜编号 | 机房内的机柜编号 | 是 | 同一机房内唯一 |
| 所属机房 | 机柜所在的机房 | 是 | |
| U 位总数 | 机柜的 U 位数量 | 是 | 必须为正整数；可由用户配置，不固定在 42U |
| 状态 | 启用或停用 | 是 | 默认启用 |
| 备注 | 补充说明 | 否 | |

### 8.3 U 位

U 位是机柜内服务器安装的垂直位置单位。产品层将 U 位作为机柜的位置属性，不将其设计为独立业务实体。

| 属性 | 含义 |
|---|---|
| U 位编号 | 从 1 到 U 位总数，每个编号在机柜内唯一 |
| 占用状态 | 空闲或已占用 |
| 占用服务器 | 当前占用该 U 位的服务器（空闲时为空） |

单个 U 位在同一时间最多被一台在架服务器占用。

### 8.4 服务器

服务器是上架到机柜中的设备。

| 业务字段 | 含义 | 必填 | 说明 |
|---|---|---|---|
| 服务器标识 | 系统内唯一标识 | 是 | 由系统生成 |
| 服务器名称 | 服务器业务名称 | 是 | 全局唯一 |
| 管理 IP | 管理网络 IP 地址 | 是 | 全局唯一 |
| 资产编号 | 企业资产编号 | 否 | 可选；若填写则全局唯一 |
| 设备类型 | 如机架式服务器、刀片、存储等 | 是 | |
| 设备高度（U 数） | 服务器占用的 U 位数量 | 是 | 必须为正整数 |
| 运行状态 | 正常、异常、维护 | 是 | 人工维护；MVP 不自动采集 |
| 所属系统 | 服务器所属的业务系统或用途 | 否 | |
| 负责人 | 服务器负责人或维护人 | 否 | |
| 备注 | 补充说明 | 否 | |

不包含 CPU、内存、硬盘、序列号、保修期、采购日期、折旧等完整资产字段。这些字段进入未来 backlog。

### 8.5 服务器位置记录

记录服务器当前或历史的位置信息。

| 业务字段 | 含义 |
|---|---|
| 服务器 | 关联的服务器 |
| 机房 | 所在机房 |
| 机柜 | 所在机柜 |
| 起始 U 位 | 占用 U 位范围的起始编号 |
| 结束 U 位 | 占用 U 位范围的结束编号 |
| 在架状态 | 未上架、在架、已下架 |
| 上架时间 | 最近一次上架操作的时间 |
| 最近位置变更时间 | 最近一次位置变更操作的时间 |

一台服务器在任意时间最多只有一个当前有效位置记录（状态为"在架"）。

### 8.6 操作记录

记录对服务器位置的所有变更操作。

| 业务字段 | 含义 |
|---|---|
| 操作类型 | 上架、移动、下架 |
| 操作对象 | 被操作的服务器 |
| 操作前机房 | 操作前所在的机房（上架时为空） |
| 操作前机柜 | 操作前所在的机柜（上架时为空） |
| 操作前起始 U 位 | 操作前起始 U 位（上架时为空） |
| 操作后机房 | 操作后所在的机房（下架时为空） |
| 操作后机柜 | 操作后所在的机柜（下架时为空） |
| 操作后起始 U 位 | 操作后起始 U 位（下架时为空） |
| 操作人 | 执行操作的用户 |
| 操作时间 | 操作执行的时间 |
| 备注 | 操作原因或说明 |

操作记录不扩展为完整审计平台。每条记录为只读，创建后不可修改或删除。

## 9. 业务状态

本章只定义当前业务所需的最小状态集合，不建立复杂状态机。

### 9.1 机房状态

| 状态值 | 含义 | 业务影响 |
|---|---|---|
| 启用 | 机房正常使用中 | 可以管理其中的机柜和服务器 |
| 停用 | 机房暂停使用 | 不能新增机柜或执行位置操作；已有数据保留可查看 |

### 9.2 机柜状态

| 状态值 | 含义 | 业务影响 |
|---|---|---|
| 启用 | 机柜正常使用中 | 可以在其中执行服务器位置操作 |
| 停用 | 机柜暂停使用 | 不能在其中执行位置操作；已有数据保留可查看 |

### 9.3 服务器位置状态

| 状态值 | 含义 | 说明 |
|---|---|---|
| 未上架 | 服务器已录入但未分配到任何机柜 U 位 | 不占用任何 U 位；可上架 |
| 在架 | 服务器当前占用指定机柜的指定 U 位范围 | 占用 U 位；可移动、可下架 |
| 已下架 | 服务器已从机柜移除 | 不占用 U 位；可重新上架 |

注意：服务器被删除不等于下架。MVP 不实现服务器删除功能；删除服务器及其影响由后续任务根据实际需求定义。

### 9.4 服务器运行状态

| 状态值 | 含义 | 说明 |
|---|---|---|
| 正常 | 据维护人了解服务器正常运行 | 人工标记 |
| 异常 | 据维护人了解服务器存在异常 | 人工标记 |
| 维护 | 服务器正在维护中 | 人工标记 |

**重要约束：** MVP 不自动采集服务器运行状态。运行状态仅作为人工维护的展示信息，不代表实时监控结果。页面上必须明确标注该状态为"人工维护"或等效说明。

## 10. 核心业务流程

### 10.1 创建机房

**前置条件：** 用户已登录且有修改权限。

**流程：**
1. 用户进入机房管理页面。
2. 用户选择"新增机房"。
3. 用户填写机房名称（必填）、位置说明（选填）、备注（选填）。
4. 用户提交保存。
5. 系统校验机房名称是否全局唯一。
6. 若校验通过，系统创建机房记录，状态默认"启用"。
7. 系统跳转或刷新至机房列表，新机房出现在列表中。
8. 若名称重复，系统返回明确错误提示"机房名称已存在"，机房不被创建。

**异常：**
- 必填字段为空：系统提示具体缺失字段。
- 名称重复：提示"机房名称已存在"。

### 10.2 创建机柜

**前置条件：** 目标机房存在且状态为"启用"；用户已登录且有修改权限。

**流程：**
1. 用户选择目标机房。
2. 用户在机房详情或机柜管理中选择"新增机柜"。
3. 用户填写机柜编号（必填）、U 位总数（必填）、备注（选填）。
4. 用户提交保存。
5. 系统校验同一机房内机柜编号是否唯一。
6. 系统校验 U 位总数是否为正整数。
7. 若校验通过，系统创建机柜记录，状态默认"启用"。
8. 若编号重复，系统返回错误提示"该机房内机柜编号已存在"。

**异常：**
- 必填字段为空：提示缺失字段。
- 编号重复：提示"该机房内机柜编号已存在"。
- U 位总数不合法（非正整数或为 0）：提示"U 位总数必须为正整数"。

### 10.3 录入服务器

**前置条件：** 用户已登录且有修改权限。

**流程：**
1. 用户进入服务器管理页面，选择"新增服务器"。
2. 用户填写服务器名称（必填）、管理 IP（必填）、设备类型（必填）、设备高度（必填），以及资产编号、所属系统、负责人、备注等选填信息。
3. 用户提交保存。
4. 系统校验名称全局唯一、管理 IP 全局唯一。
5. 若资产编号填写则校验全局唯一。
6. 若校验通过，系统创建服务器记录。
7. 服务器初始位置状态为"未上架"，不占用任何 U 位。
8. 服务器运行状态默认"正常"。

**注意事项：**
- 录入服务器与上架是独立操作。用户可以先录入服务器后，再单独执行上架操作。
- 服务器名称和管理 IP 的全局唯一性规则在本基线中通过"产品假设"章节明确（见第 17 章）。

**异常：**
- 必填字段为空：提示缺失字段。
- 名称重复：提示"服务器名称已存在"。
- 管理 IP 重复：提示"管理 IP 已存在"。
- 设备高度为非正整数：提示"设备高度必须为正整数"。

### 10.4 服务器上架

**前置条件：** 服务器存在且位置状态为"未上架"或"已下架"；目标机柜存在且状态为"启用"；用户已登录且有修改权限。

**流程：**
1. 用户选择待上架的服务器。
2. 用户选择目标机房。
3. 用户选择目标机柜。
4. 用户选择或输入起始 U 位。
5. 系统根据服务器设备高度自动计算结束 U 位：结束 U 位 = 起始 U 位 + 设备高度 - 1。
6. 系统校验：
   a. 起始 U 位 ≥ 1。
   b. 结束 U 位 ≤ 机柜 U 位总数（不超出机柜范围）。
   c. 起始 U 位到结束 U 位范围内的所有 U 位均为空闲（无其他在架服务器占用）。
7. 若所有校验通过，系统显示确认信息：服务器名称、目标机房、目标机柜、U 位范围。
8. 用户确认。
9. 系统完成上架：服务器位置状态变为"在架"；目标 U 位范围标记为已占用；更新机柜容量统计；写入操作记录（操作类型="上架"）。

**异常：**
- 目标位置超出机柜范围：提示"U 位范围超出机柜范围"，操作不生效。
- 目标位置部分或全部被占用：提示"目标 U 位已被占用"，操作不生效。
- 服务器已处于"在架"状态：提示"服务器已在架，请使用移动功能"，操作不生效。

### 10.5 服务器移动

**前置条件：** 服务器存在且位置状态为"在架"；目标机柜存在且状态为"启用"；用户已登录且有修改权限。

**流程：**
1. 用户选择当前在架的服务器。
2. 系统显示服务器当前位置：机房、机柜、起始 U 位、结束 U 位。
3. 用户选择新机房、新机柜和新起始 U 位。
4. 系统根据服务器设备高度计算新结束 U 位。
5. 系统校验新位置的合法性（同上架校验规则 a-c，但不包括服务器自身当前占用的 U 位）。若新位置与当前位置完全相同，系统提示"新位置与当前位置相同，无需移动"，不执行移动操作，不产生操作记录，不更新位置时间。
6. 若校验通过，系统显示确认信息：服务器名称、原位置、新位置。
7. 用户确认。
8. 系统完成移动：
   a. 释放原 U 位（原位置 U 位变为空闲）。
   b. 占用新 U 位（新位置 U 位标记为已占用）。
   c. 更新位置记录。
   d. 更新机柜容量统计（原机柜和新机柜均更新）。
   e. 写入操作记录（操作类型="移动"，记录操作前和操作后位置）。

**关键约束：**
- 步骤 8a 和 8b 在业务上必须是原子操作：如果占用新位置失败，原位置不能被释放。具体实现方式由架构和开发任务决定，产品基线只要求业务原子性。
- 移动失败时，服务器保持原位置不变。

**异常：**
- 新位置超出机柜范围：提示"U 位范围超出机柜范围"，原位置保持不变。
- 新位置被其他服务器占用：提示"目标 U 位已被占用"，原位置保持不变。
- 新位置与当前位置完全相同：提示"新位置与当前位置相同，无需移动"。

### 10.6 服务器下架

**前置条件：** 服务器存在且位置状态为"在架"；用户已登录且有修改权限。

**流程：**
1. 用户选择当前在架的服务器。
2. 系统显示服务器当前位置：机房、机柜、U 位范围。
3. 用户选择"下架"操作。
4. 系统显示确认信息：服务器名称、当前位置。
5. 用户确认。
6. 系统完成下架：
   a. 释放所有占用的 U 位。
   b. 服务器位置状态变为"已下架"。
   c. 服务器记录保留在系统中，不删除。
   d. 更新机柜容量统计。
   e. 写入操作记录（操作类型="下架"，操作后位置为空）。

**重要：** 下架不等于删除服务器。下架后服务器仍存在于系统中，可以重新上架。

**异常：**
- 服务器不在架：提示"服务器未在架，无法下架"。

### 10.7 服务器查询

**前置条件：** 用户已登录。

**流程：**
1. 用户进入服务器列表页面。
2. 用户输入一个或多个查询条件：
   - 服务器名称（支持模糊匹配）
   - 管理 IP（支持模糊匹配）
   - 资产编号（精确匹配）
   - 所属机房
   - 所属机柜
   - 位置状态（未上架/在架/已下架）
   - 运行状态（正常/异常/维护）
   - 所属系统（支持模糊匹配）
3. 系统返回匹配的服务器列表，至少包含：名称、管理 IP、位置状态、当前机房和机柜、运行状态。
4. 用户可点击某台服务器进入服务器详情。
5. 用户可清空条件重置列表。

**非目标：** MVP 不实现复杂全文检索、搜索引擎或高级筛选表达式。

### 10.8 查看机柜二维视图

**前置条件：** 目标机柜存在；用户已登录。

**流程：**
1. 用户从机柜列表或机房详情进入目标机柜的二维视图页面。
2. 系统显示：
   a. 机柜名称和所属机房。
   b. 所有 U 位，按 U 位编号排列，明确标注编号。
   c. 空闲 U 位和已占用 U 位有明显的视觉区分（如不同颜色或填充）。
   d. 已占用 U 位显示服务器名称，连续占用区域在视觉上合并或框选。
   e. U 位总数、已用数、空闲数和使用率。
   f. U 位编号的方向指示。
3. 用户可点击已占用的 U 位区域查看该服务器的基本信息（名称、IP、设备类型等）。
4. 用户可从该页面发起上架、移动或下架操作。

#### 10.8.1 U 位编号方向

每个机柜的二维视图必须始终显示 U 位编号。MVP 统一采用 **U 位编号从 1 开始、自上而下递增** 作为默认展示方向（即 U1 在视图顶部，U42 在视图底部）。是否支持切换方向进入待确认事项。

产品文档不涉及前端坐标、Canvas、SVG 或渲染技术。

## 11. 业务规则

以下业务规则使用 BR 编号。所有业务规则均为系统强制约束，关键校验（如 U 位占用唯一性、位置范围校验）不得被客户端绕过。

### 11.1 机房与机柜相关规则

**机房规则：**

| 编号 | 规则 | 触发场景 |
|---|---|---|
| BR-027 | 机房名称全局唯一 | 创建机房、编辑机房名称 |
| BR-028 | 机房状态为"停用"时，不能在其中新增机柜或执行服务器位置操作 | 新增机柜、上架、移动 |

**机柜规则：**

| 编号 | 规则 | 触发场景 |
|---|---|---|
| BR-001 | 同一机房内，机柜编号必须唯一 | 创建机柜、编辑机柜编号 |
| BR-002 | 机柜 U 位总数必须为正整数（≥1） | 创建机柜、编辑 U 位总数 |
| BR-003 | 机柜 U 位编号从 1 到 U 位总数连续递增 | 机柜创建后 U 位范围确定 |
| BR-029 | 机柜状态为"停用"时，不能在其中执行服务器位置操作 | 上架、移动 |

### 11.2 服务器相关规则

| 编号 | 规则 | 触发场景 |
|---|---|---|
| BR-004 | 服务器设备高度必须为正整数（≥1） | 录入服务器、编辑设备高度 |
| BR-005 | 服务器名称全局唯一 | 录入服务器、编辑服务器名称 |
| BR-006 | 管理 IP 全局唯一 | 录入服务器、编辑管理 IP |
| BR-007 | 若填写资产编号，则全局唯一 | 录入服务器、编辑资产编号 |

### 11.3 U 位占用规则

| 编号 | 规则 | 触发场景 |
|---|---|---|
| BR-008 | 服务器的结束 U 位 = 起始 U 位 + 设备高度 − 1 | 上架、移动 |
| BR-009 | 服务器占用 U 位范围不得超出机柜 U 位总数（结束 U 位 ≤ U 位总数；起始 U 位 ≥ 1） | 上架、移动 |
| BR-010 | 同一 U 位在同一时间最多被一台在架服务器占用 | 上架、移动 |
| BR-011 | 上架前必须校验目标 U 位范围全部空闲 | 上架、移动 |

### 11.4 位置管理规则

| 编号 | 规则 | 触发场景 |
|---|---|---|
| BR-012 | 一台服务器在任意时间最多有一个当前有效位置（状态为"在架"） | 上架 |
| BR-013 | 未上架或已下架的服务器不占用任何 U 位 | 上架、移动、下架 |
| BR-014 | 移动操作必须先占用新位置成功后才能释放原位置（业务原子性） | 移动 |
| BR-015 | 移动失败时，服务器保持原位置不变，原 U 位不被释放 | 移动 |
| BR-016 | 下架操作释放服务器占用的全部 U 位，服务器记录保留不删除 | 下架 |
| BR-017 | 机柜中存在在架服务器时，不能删除该机柜 | 删除机柜 |
| BR-018 | 机房中存在机柜时，不能删除该机房 | 删除机房 |
| BR-030 | 编辑机柜 U 位总数时，新值不得小于当前已用 U 数（不得导致现有在架服务器占用超出新范围） | 编辑机柜 U 位总数 |

### 11.5 容量计算规则

| 编号 | 规则 | 触发场景 |
|---|---|---|
| BR-019 | 机柜已用 U 数 = 该机柜中被在架服务器占用的 U 位总数 | 上架、移动、下架后重新计算 |
| BR-020 | 机柜空闲 U 数 = U 位总数 − 已用 U 数 | 同上 |
| BR-021 | 机柜使用率 = 已用 U 数 ÷ U 位总数 × 100%（结果保留整数或一位小数） | 同上 |

### 11.6 操作记录规则

| 编号 | 规则 | 触发场景 |
|---|---|---|
| BR-022 | 每次位置变更（上架、移动、下架）必须写入一条操作记录 | 上架、移动、下架 |
| BR-023 | 操作记录创建后不可修改或删除 | 所有位置操作 |
| BR-024 | 操作记录必须包含操作类型、操作对象、操作前后位置、操作人和操作时间 | 所有位置操作 |

### 11.7 运行状态规则

| 编号 | 规则 | 触发场景 |
|---|---|---|
| BR-025 | 服务器运行状态由人工维护，系统不自动采集或更新 | 查看或更新服务器运行状态 |
| BR-026 | 页面展示运行状态时，必须明确标注为"人工维护"或等效说明 | 所有展示运行状态的页面 |

## 13. 功能要求

每条 FR 包含编号、名称、说明、前置条件、主要规则、成功结果、失败结果和对应验收标准。

---

### FR-001：机房管理

**名称：** 机房基础信息管理

**说明：** 用户可以创建、查看和编辑机房信息。支持查看机房列表和单个机房详情。

**前置条件：** 用户已登录且有修改权限（创建和编辑）；用户已登录（查看）。

**主要规则：**
- 机房名称必填且全局唯一。
- 状态默认为"启用"。
- 机房状态为"停用"时不能在其中新增机柜或执行位置操作。

**成功结果：** 创建/编辑的机房正确保存并在列表中可查看。

**失败结果：** 名称为空或重复时给出明确提示，操作不生效。

**对应验收标准：** AC-001, AC-002, AC-021, AC-022

---

### FR-002：机柜管理

**名称：** 机柜基础信息管理

**说明：** 用户可以在指定机房内创建、查看和编辑机柜信息。机柜 U 位总数可在创建时配置。

**前置条件：** 目标机房存在且状态为"启用"；用户已登录且有修改权限。

**主要规则：**
- 同一机房内机柜编号唯一。
- U 位总数必须为正整数。
- 状态默认为"启用"。

**成功结果：** 创建/编辑的机柜正确保存并在机房详情和机柜列表中可查看。

**失败结果：** 编号重复或 U 位总数不合法时给出提示，操作不生效。

**对应验收标准：** AC-003, AC-023, AC-024

---

### FR-003：服务器基础信息录入

**名称：** 服务器基础信息录入与管理

**说明：** 用户可以录入服务器基础信息。服务器可以先录入但暂不上架。

**前置条件：** 用户已登录且有修改权限。

**主要规则：**
- 服务器名称和管理 IP 全局唯一。
- 设备高度必须为正整数。
- 资产编号若填写则全局唯一。
- 初始位置状态为"未上架"。
- 运行状态默认为"正常"（人工维护）。

**成功结果：** 服务器记录创建成功，出现在服务器列表中。

**失败结果：** 必填字段缺失或唯一性冲突时给出提示，操作不生效。

**对应验收标准：** AC-004, AC-025, AC-026, AC-027

---

### FR-004：服务器上架

**名称：** 服务器上架操作

**说明：** 用户将未上架或已下架的服务器上架到指定机柜的指定 U 位范围。系统校验 U 位可用性。

**前置条件：** 服务器位置状态为"未上架"或"已下架"；目标机柜状态为"启用"；用户已登录且有修改权限。

**主要规则：**
- 结束 U 位 = 起始 U 位 + 设备高度 − 1。
- 目标 U 位范围不能超出机柜范围。
- 目标 U 位范围必须全部空闲。
- 上架后服务器位置状态变为"在架"。
- 写入操作记录。

**成功结果：** 服务器位置状态变为"在架"，目标 U 位标记为已占用，机柜容量更新，操作记录写入。

**失败结果：** U 位不可用或超出范围时给出提示，操作不生效，服务器保持原状态。

**对应验收标准：** AC-005, AC-006, AC-007

---

### FR-005：服务器移动

**名称：** 服务器移动操作

**说明：** 用户将在架服务器移动到新机房、新机柜和新 U 位。支持跨机柜移动。

**前置条件：** 服务器位置状态为"在架"；目标机柜状态为"启用"；用户已登录且有修改权限。

**主要规则：**
- 校验新位置的合法性（不超出范围、全部空闲）。
- 移动成功后才释放原位置 U 位。
- 移动失败时原位置保持不变。
- 写入操作记录（含操作前和操作后位置）。
- 原机柜和目标机柜的容量统计均更新。

**成功结果：** 原 U 位释放，新 U 位被占用，位置记录更新，操作记录写入。

**失败结果：** 新位置不合法时给出提示，原位置不变。

**对应验收标准：** AC-008, AC-009

---

### FR-006：服务器下架

**名称：** 服务器下架操作

**说明：** 用户将在架服务器从机柜中移除，释放 U 位但保留服务器记录。

**前置条件：** 服务器位置状态为"在架"；用户已登录且有修改权限。

**主要规则：**
- 释放所有占用的 U 位。
- 服务器位置状态变为"已下架"。
- 服务器记录保留在系统中。
- 写入操作记录。

**成功结果：** 服务器位置状态变为"已下架"，U 位释放，服务器记录仍存在，操作记录写入。

**失败结果：** 服务器不在架时给出提示，操作不生效。

**对应验收标准：** AC-010, AC-011

---

### FR-007：服务器查询

**名称：** 服务器查询与筛选

**说明：** 用户可以通过服务器名称、管理 IP、资产编号、所属机房、所属机柜、位置状态、运行状态和所属系统等条件查询服务器。

**前置条件：** 用户已登录。

**主要规则：**
- 支持名称、IP、所属系统模糊匹配。
- 支持机房、机柜、位置状态、运行状态精确筛选。
- 查询结果至少显示名称、IP、位置状态、当前机房和机柜、运行状态。

**成功结果：** 返回匹配条件的服务器列表。

**失败结果：** 无匹配结果时显示空列表，不报错。

**对应验收标准：** AC-012, AC-028

---

### FR-008：二维机柜 U 位视图

**名称：** 机柜二维 U 位视图展示

**说明：** 以二维视图展示单个机柜的全部 U 位，区分空闲和已占用状态，显示服务器名称和容量统计。

**前置条件：** 目标机柜存在；用户已登录。

**主要规则：**
- 显示所有 U 位编号。
- 空闲和已占用 U 位有明显视觉区分。
- 已占用区域显示服务器名称和起止 U 位。
- 连续 U 位占用在视觉上连续。
- 显示 U 位总数、已用数、空闲数和使用率。
- U 位编号方向统一为自上而下递增（U1 在顶部）。
- 点击已占用区域可查看服务器基本信息。

**成功结果：** 视图正确展示所有 U 位状态和容量统计，与实际数据一致。

**失败结果：** 数据加载失败时显示适当提示。

**对应验收标准：** AC-013, AC-014, AC-029

---

### FR-009：机柜容量统计

**名称：** 机柜容量和使用情况展示

**说明：** 系统计算并展示每个机柜的 U 位总数、已用数、空闲数和使用率。

**前置条件：** 机柜存在且有机柜 U 位总数和服务器占用数据。

**主要规则：**
- 已用 U 数 = 该机柜中所有在架服务器占用的 U 位总数。
- 使用率 = 已用 U 数 ÷ U 位总数 × 100%。
- 容量在每次上架、移动、下架操作后立即更新。

**成功结果：** 机柜列表和二维视图中展示的容量数据与实际占用一致。

**失败结果：** N/A：容量计算为系统自动执行。

**对应验收标准：** AC-014

---

### FR-010：操作记录查看

**名称：** 位置变化操作记录查看

**说明：** 用户可以按服务器查看其位置变化操作记录。

**前置条件：** 服务器存在且至少有一次位置操作；用户已登录。

**主要规则：**
- 操作记录包含操作类型、操作前后位置、操作人、操作时间、备注。
- 操作记录按时间倒序排列。
- 操作记录为只读，不可修改或删除。

**成功结果：** 页面展示服务器的完整位置变化历史。

**失败结果：** 无操作记录时显示空列表，不报错。

**对应验收标准：** AC-015

---

### FR-011：人工运行状态展示

**名称：** 服务器人工运行状态展示

**说明：** 系统展示服务器运行状态（正常/异常/维护），但该状态由人工维护，非实时采集。

**前置条件：** 服务器记录存在。

**主要规则：**
- 运行状态在服务器列表和详情中展示。
- 运行状态由具有修改权限的用户手动更新。
- 展示运行状态的页面必须标注该状态为人工维护，非实时监控数据。

**成功结果：** 运行状态正确展示且标注人工维护属性。

**失败结果：** N/A：展示功能无失败路径；若状态更新操作失败则以明确提示告知用户。

**对应验收标准：** AC-016, AC-030

---

### FR-012：基本信息变更约束

**名称：** 基本信息变更防误操作约束

**说明：** 对已有关联数据的核心对象实施基本的变更保护。

**前置条件：** 目标对象存在且有关联数据。

**主要规则：**
- 机柜中存在在架服务器时，不能删除该机柜。
- 机房中存在机柜时，不能删除该机房。
- 编辑操作（如修改机柜 U 位总数）需要校验不会导致现有服务器占用超限。

**成功结果：** 受保护的删除操作被系统拒绝并给出明确原因。

**失败结果：** N/A：这是防御性约束。

**对应验收标准：** AC-017, AC-031

## 14. 非功能要求

### NFR-001：数据一致性——U 位占用

**说明：** 系统必须保证同一 U 位在同一时间最多被一台在架服务器占用。任何位置操作不得产生部分完成的状态（要么全部成功，要么全部失败，不产生中间状态）。

**对应验收标准：** AC-005, AC-006, AC-009

---

### NFR-002：基础操作响应时间

**说明：** 在基准数据规模下（单个机房 ≤ 200 个机柜，单个机柜 ≤ 60 U，服务器总数 ≤ 10000），使用有线局域网连接（客户端到服务端网络延迟 ≤ 5ms，带宽 ≥ 100 Mbps），从页面请求发起（浏览器导航或点击链接）到首屏数据完整显示（列表数据或二维视图渲染完毕，不含用户交互时间），目标页面应在 3 秒内完成加载。具体计时口径（服务端处理时间、网络传输时间、客户端渲染时间的测量方式）由架构任务在技术验证方案中定义，但 3 秒阈值不得单方面调整。

**对应验收标准：** AC-018

---

### NFR-003：二维视图可用性

**说明：** 机柜二维视图中，空闲 U 位和已占用 U 位必须在使用色觉障碍友好（不仅依赖颜色）的方式下清晰可辨。建议使用符号、纹理或标注辅助区分。

**对应验收标准：** AC-013

---

### NFR-004：位置操作校验反馈

**说明：** 服务器上架、移动和下架操作中，如果目标位置不可用（超出范围、已被占用），系统必须给出明确、可理解的错误提示，提示中包含具体的失败原因（如"U 位 10-11 已被服务器 X 占用"）。

**对应验收标准：** AC-006, AC-008

---

### NFR-005：操作可追踪性

**说明：** 每次服务器位置变更（上架、移动、下架）必须产生一条包含操作类型、操作对象、操作前后位置、操作人和操作时间的操作记录。记录创建后不可修改或删除。

**对应验收标准：** AC-015

---

### NFR-006：浏览器兼容性

**说明：** 系统必须在企业常用桌面浏览器上正常运行。MVP 支持验收时最新的稳定版 Chrome、Edge 和 Firefox 桌面版（验收记录中应注明具体浏览器及版本号）。不要求移动端浏览器适配。

**对应验收标准：** AC-019

---

### NFR-007：基本操作安全

**说明：** MVP 要求所有页面访问均需登录身份，不实现匿名访问。所有修改操作（创建、编辑、上架、移动、下架）仅限具有修改权限的角色（机房管理员、运维人员）执行。只读用户（DBA/应用运维人员、只读查看人员）尝试执行修改操作时，系统必须拒绝。

**对应验收标准：** AC-020

---

## 17. 产品假设

以下假设为产品经理在信息不完整时做出的最小、可逆判断。假设变化时应评估对关联 FR 的影响。

| 编号 | 假设内容 | 假设依据 | 若假设变化可能影响的要求 | 风险等级 |
|---|---|---|---|---|
| AS-01 | 服务器名称全局唯一 | 避免查询和操作时混淆不同服务器；企业运维实践中服务器名称通常唯一 | FR-003, FR-007 | 低 |

| AS-08 | 用户主要通过桌面浏览器访问系统 | 机房运维人员通常在固定工位操作 | NFR-006 | 低 |

## 21. 后续任务输入

产品基线完成后，建议依次建立以下任务。这些仅为产品层建议，具体任务拆分和优先级由 Architect 决定。

### 21.1 建议后续任务

1. **系统架构基线任务**：定义系统整体架构、前后端技术选型、模块边界、数据流和部署方式。
2. **业务数据模型任务**：基于本基线的业务对象，设计数据库物理模型（表结构、字段类型、索引、关系）。
3. **API 契约任务**：基于本基线的 FR 和业务流程，设计 REST API 或 API 契约。
4. **低保真 UI 任务**：基于本基线的页面清单和二维视图要求，设计线框图或低保真原型。
5. **前端脚手架任务**：搭建 Vue 3 前端项目、路由和基础组件结构。
6. **后端脚手架任务**：搭建 .NET 后端项目、数据库迁移和基础 API 结构。

### 21.2 禁止在产品基线中预定义

以下内容不得出现在本产品基线中：
- 具体数据库建表 SQL
- API 路径和方法签名
- C# 类、接口或命名空间
- Vue 组件树、props 或 slots
- Three.js 场景、相机或渲染配置
- 部署拓扑图或服务器配置
- 代码目录结构设计
